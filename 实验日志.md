## PubMedQA

- 数据来源：官方仓库 `https://github.com/pubmedqa/pubmedqa`；HF 版本 `qiaojin/PubMedQA`。
- 本地路径：`~/datasets/PubMedQA/pubmedqa_official/`。
- 数据集配置：`configs/datasets/pubmedqa.json`。
- 许可证：官方仓库与 HF dataset card 均标记 MIT。
- canonical split/file：官方 `PQA-L test_set`，`canonical/test.jsonl`。
- expected rows：`500`。
- 版本与 split：HF configs 为 `pqa_artificial`、`pqa_labeled`、`pqa_unlabeled`，均为 `train` split。`pqa_labeled` 共 `1000` 条；官方 `split_dataset.py` 使用 `random.seed(0)` 切成 `500` 条 CV 和 `500` 条 test。本地 `test_set.json` 已核对与 `test_ground_truth.json` 完全一致。
- 不使用版本：`pqa_artificial` 是机器生成标签训练集，不作为正式 benchmark；`pqa_unlabeled` 无 `final_decision`，不能算 accuracy；PQA-L CV 半区不进入正式 test 口径。
- 字段映射：原始字段含 PMID key、`QUESTION`、`CONTEXTS`、`LABELS`、`MESHES`、`YEAR`、`reasoning_required_pred`、`reasoning_free_pred`、`final_decision`、`LONG_ANSWER`；规范化为 `question`、`options`、`answer_idx`、`metadata`。
- 答案映射：`yes -> A`、`no -> B`、`maybe -> C`；推理时仍按 `shuffle_options_seed=123` 统一打乱并重映射答案。
- prompt 口径：PubMedQA 需要 abstract context；规范化时把 `QUESTION` 和带 section label 的 `CONTEXTS` 合并进 `question` 字段，不改 `infer.py` 统一 prompt。选项固定为 yes/no/maybe 三分类。
- 结构统计：官方 test set `500` 行；标签分布 `yes=276`、`no=169`、`maybe=55`；英文。
- 当前状态：主表 10 个模型均有 500 行正式输出，路径统一为 `Outputs/pubmedqa/<model-name>/<model-name>.jsonl`。
